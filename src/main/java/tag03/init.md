# Projektinitialisierung – Der Start jedes Projekts

## 1. Was ist die Projektinitialisierung?

Die **Projektinitialisierung** ist die **erste Phase** im Projektmanagement.  
Sie klärt die grundlegende Frage:  

> **„Soll das Projekt überhaupt durchgeführt werden?“**

Ziel ist es, eine **fundierte Entscheidungsgrundlage** für Auftraggeber und Projektleiter zu schaffen.

---

## 2. Warum ist die Projektinitialisierung wichtig?

| Aspekt | Bedeutung |
|--------|-----------|
| **Risikominimierung** | Frühzeitig erkennen, ob das Projekt sinnvoll / machbar ist |
| **Klarheit** | Wer macht was, warum und mit welchen Mitteln? |
| **Vermeidung von Fehlinvestitionen** | Bevor Kosten und Ressourcen gebunden werden |
| **Entscheidungsgrundlage** | Ja / Nein – oder erst später starten |

> Ohne saubere Initialisierung starten viele Projekte mit unklaren Zielen → hohes Risiko.

---

## 3. Typische Schritte der Projektinitialisierung

| Schritt | Beschreibung |
|---------|--------------|
| 1. **Projektidee / Anstoß** | Erste Idee, Problem, Kundenanfrage, gesetzlicher Auftrag |
| 2. **Machbarkeitsstudie** | Ist das Projekt technisch, wirtschaftlich, personell machbar? |
| 3. **Projektauftrag** (formell) | Dokument, das das Projekt offiziell freigibt |
| 4. **Projektleiter benennen** | Wer führt das Projekt? |
| 5. **Projektteam zusammenstellen** | Erste Kernmitglieder |
| 6. **Projektziele definieren** | Was soll erreicht werden? (SMART) |
| 7. **Rahmenbedingungen klären** | Budget, Termine, Qualität, Ressourcen |
| 8. **Auftraggeber freigabe** | Entscheidung über Projektstart |

---

## 4. Wichtige Dokumente in der Initialisierung

| Dokument | Inhalt |
|----------|--------|
| **Projektskizze** | Grobe Beschreibung von Ziel, Nutzen, Risiken, Kosten |
| **Machbarkeitsstudie** | Technische, wirtschaftliche, organisatorische Machbarkeit |
| **Projektauftrag** | Verbindliche Freigabe, enthält Ziele, Budget, Termine, Kompetenzen |
| **Lastenheft** | Was der Auftraggeber fordert (nicht wie) |
| **Pflichtenheft** | Wie das Projektteam die Anforderungen umsetzt |

---

## 5. Wer ist in der Initialisierung beteiligt?

| Rolle | Aufgabe |
|-------|---------|
| **Auftraggeber** | Gibt Projektidee vor, entscheidet über Freigabe |
| **Projektleiter** | Bereitet Entscheidungsgrundlagen vor |
| **Fachabteilung** | Liefert erste Anforderungen, prüft Machbarkeit |
| **Projektmanagement‑Office (PMO)** | Unterstützt bei Methodik, Vorlagen |
| **Lenkungsausschuss** | Freigabe bei großen Projekten |

---

## 6. Ergebnis der Projektinitialisierung

Nach erfolgreicher Initialisierung liegt vor:

- ✅ Entscheidung: **Projekt wird gestartet** (oder abgelehnt / zurückgestellt)
- ✅ **Projektauftrag** liegt unterschrieben vor
- ✅ **Projektleiter** und **erstes Kernteam** sind bekannt
- ✅ **Grobe Ziele, Termine, Budget** sind definiert

---

## 7. Typische Fehler in der Initialisierung

| Fehler | Folge |
|--------|-------|
| Unklare Ziele | Projekt wird zum „Dauerläufer“ |
| Fehlende Machbarkeitsprüfung | Spätere technische / finanzielle Probleme |
| Keine Auftraggeber-Freigabe | Projekt läuft ohne Rückhalt |
| Zu viele Beteiligte | Lange Entscheidungswege |
| Keine Dokumentation | Keine Nachvollziehbarkeit, später Streit |

---

## 8. Prüfungsrelevanz (IHK AP1)

| Häufige Fragen | Antwort |
|----------------|---------|
| Wann findet die Projektinitialisierung statt? | Ganz am Anfang – **vor** der Planung |
| Was ist das Ergebnis? | **Projektauftrag** (freigabe) |
| Wer gibt das Projekt frei? | Der **Auftraggeber** |
| Was passiert bei Nicht‑Machbarkeit? | Projekt wird **nicht gestartet** (abgelehnt / zurückgestellt) |
| Welches Dokument beschreibt „Was“? | **Lastenheft** (Auftraggebersicht) |

---

## 9. Merksätze für die Prüfung

> **„Initialisierung klärt das ‚Ob‘ – erst dann kommt das ‚Wie‘.“**

> **„Ohne Projektauftrag kein Projekt.“**

> **„Der Auftraggeber entscheidet – der Projektleiter bereitet vor.“**

> **„Machbarkeit vor Start – sonst kostet es nur Geld.“**

---

## 10. Zusammenfassung als Checkliste

| Schritt | Erledigt? |
|---------|-----------|
| Projektidee ist dokumentiert | ☐ |
| Machbarkeitsstudie durchgeführt | ☐ |
| Projektauftrag erstellt | ☐ |
| Auftraggeber hat freigegeben | ☐ |
| Projektleiter benannt | ☐ |
| Erste Ziele definiert (SMART) | ☐ |
| Grober Zeit‑ / Kostenrahmen festgelegt | ☐ |
