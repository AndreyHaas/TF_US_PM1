## Rollen in einem Projekt – basierend auf den drei Fragen:

---

### 1. Wer hat Einfluss auf das Projekt?  
👉 **Auftraggeber / Stakeholder / Fachbereich**  
(z. B. Kunde, Projektinitiator, Nutzervertreter, Finanzabteilung, Aufsichtsrat)

---

### 2. Wer setzt das Projekt um?  
👉 **Projektteam / Entwickler / Umsetzer**  
(z. B. Softwareentwickler, Tester, Datenbankadministratoren, UX‑Designer, Dokumentierer)

---

### 3. Wer steuert das Projekt?  
👉 **Projektleiter / Projektmanagement / Lenkungsausschuss**  
(z. B. Projektleiter, Teilprojektleiter, Projektmanagement‑Office (PMO))

---

## Typische Projektrollen (klassisch / IHK‑gerecht)

| Rolle | Aufgaben |
|--------|-----------|
| **Auftraggeber** | finanziert, entscheidet, gibt Ziele vor, nimmt ab |
| **Lenkungsausschuss** | strategische Entscheidungen, Änderungsfreigaben |
| **Projektleiter** | plant, steuert, kontrolliert, berichtet |
| **Projektteam** | setzt fachlich / technisch um |
| **Fachbereich** | liefert Fachinhalte, testet, priorisiert |
| **Anwender (Nutzer)** | nutzt später das Ergebnis, gibt Feedback |

---

## Merksatz für die Prüfung

> *„Einfluss haben Auftraggeber, Lenkungskreis und Fachbereich.  
> Umsetzung macht das Projektteam.  
> Steuerung übernimmt der Projektleiter.“* ✅
